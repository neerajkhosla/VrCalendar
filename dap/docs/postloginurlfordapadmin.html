Normally if you - the DAP Admin - first log in to WordPress Admin, then click on the "Digital Access Pass" link in the WordPress admin menu on the left, you will be redirected to the DAP Admin Dashboard.

But let's say you have set "Auto-login ALL WordPress Admins as DAP Admin?" to "No".

Now when you click on the "Digital Access Pass" link in the WordPress admin menu on the left, you will be presented with the default DAP login form (http://YourSite.com/dap/login.php). You may have done this because you don't want your staff or other WordPress admins to access the DAP admin dashboard. So when you enter the DAP admin's email and password on this login form, since you are both a DAP admin as well as one of the users in your system, where should DAP redirect you to?

1) Do you wish to be redirected to the DAP Admin Dashboard (just because you are the DAP Admin)?

OR...

2) Do you wish to be redirected to your regular member's area as if you are just another user?

That's what this field allows you to configure.

So, this is the URL to which the DAP Admin user is redirected to, right after they log in, if s/he logs in directly from a DAP login form on your web site.

Summary

1) If you set this to "DAP Admin Home", then you will be redirected to the DAP Admin Dashboard.

2) If you set this to "Member Post-Login URL", then you will be redirected to your regular member's area as if you are just another user.
